    Referência de API

    Documentação completa da API para todos os módulos do Brazilian Utils.

    Módulos

    Estados

    Trabalhe com estados brasileiros (UFs).

    Funções Principais: - valid-uf? - Valide código de estado (keyword) - uf->state-name - Obtenha nome completo do estado - uf->area-codes - Obtenha códigos de área (DDDs) - uf->ie-length - Obtenha tamanho do registro IE - uf->code - Obtenha código numérico do estado - all-ufs - Obtenha todos os 27 estados - all-state-names - Obtenha nomes de todos os estados

    Cidades

    Acesse dados de cidades brasileiras organizadas por estado.

    Funções Principais: - cities-of - Obtenha cidades de um estado (retorna vetor vazio se inválido) - cities-of! - Obtenha cidades ou lance erro se estado inválido - all-cities - Obtenha todas as cidades com seus estados associados - all-city-names - Obtenha lista plana de todos os nomes de cidades - find-city-by-name - Procure cidades por nome (case-insensitive) - city-exists? - Verifique se uma cidade existe em um estado

    CEP

    Valide, formate e busque informações de códigos postais brasileiros (CEP).

    Funções Principais: - is-valid? - Valide formato CEP (8 dígitos) - remove-symbols - Remova hífens e caracteres não-numéricos - format-cep - Formate CEP com hífen (XXXXX-XXX) - get-address-from-cep - Busque endereço usando ViaCEP API - get-cep-information-from-address - Busque CEP usando endereço

    CPF

    Valide, formate e gere números de CPF (Cadastro de Pessoa Física).

    Funções Principais: - is-valid? - Valide CPF (11 dígitos, check digits corretos) - remove-symbols - Remova pontos, hífens e espaços - format-cpf - Formate CPF com máscara (XXX.XXX.XXX-XX) - generate - Gere CPF válido aleatório (com UF opcional)

    Recursos: - Suporta validação com ou sem formatação - Detecta números reservados (todos dígitos iguais) - Gera CPF com estado específico se desejado

    CNPJ

    Valide, formate e gere números de CNPJ (Cadastro Nacional de Pessoa Jurídica).

    Funções Principais: - is-valid? - Valide CNPJ (14 dígitos, numérico ou alfanumérico) - remove-symbols - Remova pontos, barras, hífens - format-cnpj - Formate CNPJ com máscara (XX.XXX.XXX/XXXX-XX) - generate - Gere CNPJ numérico válido aleatório - generate-alfanumeric - Gere CNPJ alfanumérico válido (conforme IN RFB nº 2.229/2024)

    Recursos: - Valida CNPJ numérico e alfanumérico - Suporta formatação com ou sem pontuação - Detecta números com dígitos repetidos

    PIS

    Valide números de PIS (Programa de Integração Social).

    Funções Principais: - is-valid? - Valide PIS (11 dígitos) - remove-symbols - Remova pontos, hífens

    Algoritmo: - Utiliza mod-11 com pesos [3 2 9 8 7 6 5 4 3 2] - Se resultado é 10 ou 11, dígito verificador é 0

    CNH

    Valide números de CNH (Carteira Nacional de Habilitação).

    Funções Principais: - is-valid? - Valide CNH (11 dígitos) - remove-symbols - Remova pontos, hífens

    Algoritmo: - Utiliza mod-11 com pesos decrescentes - Verifica dois dígitos verificadores com regras especiais

    RENAVAM

    Valide números de RENAVAM (Registro Nacional de Automóvel).

    Funções Principais: - is-valid? - Valide RENAVAM (9 ou 11 dígitos) - remove-symbols - Remova pontos, hífens

    Recursos: - Suporta RENAVAM antigo (9 dígitos) e novo (11 dígitos) - Normaliza automaticamente com zeros à esquerda quando necessário - Utiliza mod-11 com pesos específicos

    Título Eleitoral

    Valide e gere números de Título Eleitoral (Voter ID).

    Funções Principais: - is-valid? - Valide Título Eleitoral (12 dígitos) - remove-symbols - Remova pontos, hífens - validation-errors - Obtenha detalhes de erro de validação - generate - Gere Título Eleitoral válido (com UF opcional) - get-uf-code - Obtenha código TSE de um estado

    Recursos: - Utiliza códigos TSE (diferentes de códigos IBGE) - Gera check digits usando pesos específicos - Detecta números com dígitos repetidos

    Inscrição Estadual

    Valide números de Inscrição Estadual (IE) por estado.

    Funções Principais: - is-valid? - Valide IE (valida usando regras do estado) - format-ie - Formate IE conforme padrão do estado

    Recursos: - Suporta todos os 27 estados brasileiros - Cada estado tem algoritmo de validação específico - Alguns estados aceitam múltiplos formatos (ex: Bahia aceita 8 ou 9 dígitos)

    Email

    Valide endereços de email.

    Funções Principais: - is-valid? - Valide formato de email

    Recursos: - Validação básica de formato

    Telefone

    Valide números de telefone brasileiros.

    Funções Principais: - is-valid? - Valide telefone (celular ou fixo) - is-valid-mobile? - Valide apenas celular - is-valid-landline? - Valide apenas telefone fixo - validation-errors - Obtenha detalhes de erro - mobile-validation-errors - Detalhes de erro para celular - landline-validation-errors - Detalhes de erro para fixo

    Recursos: - Suporta formatos com e sem formatação - Valida DDD (código de área) - Diferencia celular (9 dígitos) de fixo (8 dígitos)

    Capitalização

    Capitalize strings respeitando regras do português brasileiro.

    Funções Principais: - capitalize - Capitalize string com regras de preposições e acrônimos

    Recursos: - Preserva palavras-chave (de, da, do, etc.) - Trata acrônimos (EU, USA, etc.) - Case-insensitive em entrada

    Moeda

    Formate e analise valores em Real Brasileiro (BRL).

    Funções Principais: - format-currency - Formate número como BRL (1.234,56) - parse-currency - Parse string BRL para número

    Recursos: - Suporta formatação de centavos - Converte entre vírgula e ponto decimal - Adiciona separador de milhares

    Placa de Veículo

    Valide e converta placas de veículos brasileiros.

    Funções Principais: - is-valid? - Valide placa (tradicional ou Mercosul) - get-format - Obtenha formato da placa (LLLNNNN ou LLLNLNN) - convert-to-mercosul - Converta placa tradicional para Mercosul

    Recursos: - Suporta formato tradicional (3 letras + 4 dígitos) - Suporta formato Mercosul (3 letras + 1 dígito + 1 letra + 2 dígitos) - Conversão automática entre formatos

    Processo Jurídico

    Valide e formate números de processo jurídico.

    Funções Principais: - is-valid? - Valide número de processo (20 dígitos) - remove-symbols - Remova pontos, hífens - format-processo - Formate com padrão (NNNNNNN-DD.AAAA.J.TT.OOOO)

    Algoritmo: - MOD 97-10 para validação de check digits - Suporta processos judiciais de diferentes segmentos

    Compatibilidade de Plataforma

    Todos os módulos funcionam perfeitamente em:

    • Clojure - Suporte completo JVM (1.11.1+)
    • ClojureScript - Navegador e Node.js (1.11.60+)

    Use condicionais de leitura #?() quando código específico de plataforma for necessário.

    #?(:clj  (clj-http.client/get url)
   :cljs (cljs-http.client/get url))

    Tratamento de Erros

    Falhas Elegantes

    A maioria das funções retorna false ou nil para entrada inválida:

    (states/valid-uf? :XX)           ; => false
(states/uf->state-name :XX)      ; => nil
(cep/is-valid? "inválido")       ; => false
(cities/cities-of :XX)           ; => [] (vetor vazio)

    Erros Explícitos

    Funções com sufixo ! lançam exceções:

    (cities/cities-of! :XX)  ; => lança ExceptionInfo com {:state :XX}

    Detalhamento de Erros

    Funções com sufixo -errors retornam vetor com mensagens:

    (phone/validation-errors "123")
; => ["Invalid area code" "Invalid phone number length"]

    Funções de Utilidade (Helpers)

    Disponíveis em brazilian-utils.helpers

    • only-numbers - Extrai apenas dígitos de string
    • repeated-digits? - Verifica se todos dígitos são iguais
    • char->digit - Converte caractere numérico para int
    • weighted-sum - Calcula soma ponderada
    • check-digit - Calcula dígito verificador genérico
    • random-digits - Gera string com dígitos aleatórios
    • http-get - Faz requisição HTTP GET (com fallback para JS)
    • safe-call - Executa função com tratamento de erro
    • parse-int - Parse seguro de string para inteiro
    • split-and-rejoin - Insere caracteres em posições específicas
    • mod-large-number - Modulo para números muito grandes (BigInt)
    • calculate-check-digit - Calcula dígito verificador genérico com regra customizada

    Exemplos de Início Rápido

    Verificar um Estado

    (require '[brazilian-utils.states :as states])

(states/valid-uf? :SP)           ; => true
(states/uf->state-name :SP)      ; => "São Paulo"
(states/uf->area-codes :SP)      ; => [11 12 13 ...]

    Validar CPF

    (require '[brazilian-utils.cpf :as cpf])

(cpf/is-valid? "123.456.789-09")  ; => true/false
(cpf/format-cpf "12345678909")    ; => "123.456.789-09"
(cpf/generate)                    ; => "12345678909" (válido)
(cpf/generate {:uf-code :SP})     ; => CPF com SP como 9º dígito

    Validar CNPJ

    (require '[brazilian-utils.cnpj :as cnpj])

(cnpj/is-valid? "12.345.678/0001-95")  ; => true/false
(cnpj/format-cnpj "12345678000195")    ; => "12.345.678/0001-95"
(cnpj/generate)                        ; => "12345678000195" (válido)
(cnpj/generate-alfanumeric)            ; => "AB1234567000195" (alfanumérico)

    Buscar CEP

    (require '[brazilian-utils.cep :as cep])

(cep/is-valid? "01310-100")                    ; => true
(cep/format-cep "01310100")                    ; => "01310-100"
(cep/get-address-from-cep "01310-100")        ; => {:logradouro "..." :localidade "..."}

    Trabalhar com Cidades

    (require '[brazilian-utils.cities :as cities])

(cities/cities-of :SP)                      ; => ["São Paulo" "Campinas" ...]
(cities/find-city-by-name "São Paulo")      ; => [{:state :SP :city "São Paulo"}]
(cities/city-exists? :SP "São Paulo")       ; => true

    Validar Telefone

    (require '[brazilian-utils.phone :as phone])

(phone/is-valid? "(11) 99999-9999")         ; => true
(phone/is-valid-mobile? "11999999999")      ; => true
(phone/validation-errors "123")             ; => ["Invalid area code" ...]

    Fontes de Dados

    • Estados - Definido em src/brazilian_utils/data.cljc
    • Cidades - Carregado de resources/cities.edn
    • CEP - Validado via ViaCEP API
    • Feriados - Obtidos via API externa (com fallback local)

    ### Validar um Endereço

```clojure
(require '[brazilian-utils.cep :as cep]
         '[brazilian-utils.cities :as cities])

(let [cep-input "01310-100"
      city-input :SP]
  (and (cep/is-valid? cep-input)
       (seq (cities/cities-of city-input))))
; => true

    Analisar um Pagamento

    (require '[brazilian-utils.boleto :as boleto])

(boleto/parse-boleto "34195.17515 23456.787128 34123.456005 5 10318000002603")
; => {:bank-code "341" :bank-name "Banco Itaú" ...}
